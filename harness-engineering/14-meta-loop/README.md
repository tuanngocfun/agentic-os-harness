# Red/Blue Meta Loop

The meta loop keeps feature work, adversarial review, patching, and evidence synchronized. A green gate is scoped evidence, never a broad security claim.

## Loop

1. **Observe**: collect compiler, runtime, IDE, static-analysis, or review symptoms.
2. **Reproduce**: preserve a deterministic failing command, marker, or historical source witness.
3. **Classify**: assign a stable red-team ID, subsystem, impact, and severity.
4. **Attack**: run the smallest guest-only or tooling-only probe that proves the assumption can fail.
5. **Patch**: implement one ownership-bound fix without weakening the reproduction.
6. **Verify**: run the targeted blue gate, functional regression gates, and adversarial gate.
7. **Release**: publish a patch note with exact evidence and explicit residual limits.
8. **Feed back**: update the attack catalog, playbook, harness profile, and contract checker.

## Executable Gates

- `make test-qemu-preflight`: rejects missing or altered BIOS data, unexpected QEMU versions, early exit, empty output, stale logs, and absent exact markers.

- `make test-static-analysis`: strict syntax for default and all selftest branches, correctness-focused cppcheck, shell syntax, editor JSON, and header provenance.
- `make test-red-team-tooling`: reproduces host-header shadowing and the historical address-space oracle weakness, then proves their blue controls.
- `make test-red-team`: runs guest-only attack/defense probes in QEMU.
- `make test-meta-loop`: runs tooling red-team, guest red-team, and the harness contract.

## Evidence Boundaries

- Editor diagnostics are not compiler findings until the cross compiler reproduces them.
- A selftest marker is not sufficient unless the test verifies the state transition it names.
- Historical reproductions remain useful after a patch, but current-source and runtime checks must prove the control.
- Static analysis supplements runtime tests; it does not prove memory safety, isolation, or production readiness.
- Machine findings belong under `build/red-team/` and are regenerated by gates.
- QEMU execution provenance belongs in `build/qemu-runtime.jsonl`; negative preflight unit tests must not manufacture runtime records.
