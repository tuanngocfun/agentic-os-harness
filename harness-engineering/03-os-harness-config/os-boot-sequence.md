# OS Boot Sequence — x86 Bare Metal

## Tổng quan

Đây là boot chain cho x86 bare metal OS từ BIOS đến `kernel_main()`. Tài liệu này giả định bootloader đơn giản: BIOS load sector 0, bootloader load raw `kernel.bin` vào `0x1000`, rồi chuyển sang protected mode và gọi kernel entry.

## Boot Chain

```
BIOS POST
  -> BIOS loads 512-byte boot sector to 0x7C00
  -> boot.asm starts in 16-bit real mode
  -> initialize DS/ES/SS/SP explicitly
  -> init COM1 serial for automated markers
  -> enable A20
  -> read raw kernel.bin sectors from disk to 0x1000 (phase-1 track-rolling CHS loader: max 120 sectors)
  -> setup GDT
  -> switch to 32-bit protected mode
  -> setup stack
  -> emit BOOT_OK on COM1
  -> jump/call kernel entry at 0x1000
  -> entry.asm calls kernel_main()
  -> kernel_main() initializes serial and emits KERNEL_INIT_OK
```

## Chi tiết từng bước

### 1. BIOS POST

- CPU reset -> BIOS code tại `0xFFFF0`.
- BIOS kiểm tra hardware cơ bản.
- BIOS load boot sector từ disk sector 0 vào `0x7C00`.
- Boot sector phải là flat 512-byte binary và kết thúc bằng signature `0xAA55`.

### 2. Boot Sector (`boot.asm`)

```nasm
[bits 16]
[org 0x7C00]

KERNEL_OFFSET equ 0x1000
COM1 equ 0x3F8

%include "boot_config.inc" ; generated by Makefile: KERNEL_SECTORS equ <n>

%if KERNEL_SECTORS > 120
%error "CHS track-rolling loader supports at most 120 kernel sectors; use LBA or a 2-stage loader"
%endif

start:
    cli
    xor ax, ax
    mov ds, ax
    mov es, ax
    mov ss, ax
    mov sp, 0x7C00
    sti

    mov [BOOT_DRIVE], dl

    call serial_init_16
    call enable_a20
    call load_kernel
    call switch_to_pm

    jmp $

serial_init_16:
    mov dx, COM1 + 1
    mov al, 0x00
    out dx, al
    mov dx, COM1 + 3
    mov al, 0x80
    out dx, al
    mov dx, COM1 + 0
    mov al, 0x03
    out dx, al
    mov dx, COM1 + 1
    mov al, 0x00
    out dx, al
    mov dx, COM1 + 3
    mov al, 0x03
    out dx, al
    mov dx, COM1 + 2
    mov al, 0xC7
    out dx, al
    mov dx, COM1 + 4
    mov al, 0x0B
    out dx, al
    ret

serial_putc_16:
    push dx
    push ax
.wait:
    mov dx, COM1 + 5
    in al, dx
    test al, 0x20
    jz .wait
    pop ax
    mov dx, COM1
    out dx, al
    pop dx
    ret

serial_puts_16:
    lodsb
    test al, al
    jz .done
    call serial_putc_16
    jmp serial_puts_16
.done:
    ret

enable_a20:
    in al, 0x92
    or al, 2
    out 0x92, al
    ret

load_kernel:
    mov bx, KERNEL_OFFSET
    mov ah, 0x02
    mov al, KERNEL_SECTORS
    mov ch, 0
    mov cl, 2
    mov dh, 0
    mov dl, [BOOT_DRIVE]
    int 0x13
    jc disk_error
    ret

disk_error:
    mov si, MSG_BOOT_DISK_ERROR
    call serial_puts_16
    jmp $

switch_to_pm:
    cli
    lgdt [gdt_descriptor]
    mov eax, cr0
    or eax, 1
    mov cr0, eax
    jmp CODE_SEG:init_pm

[bits 32]
init_pm:
    mov ax, DATA_SEG
    mov ds, ax
    mov ss, ax
    mov es, ax
    mov fs, ax
    mov gs, ax

    mov ebp, 0x90000
    mov esp, ebp

    mov esi, MSG_BOOT_OK
    call serial_puts_32

    call KERNEL_OFFSET
    jmp $

serial_putc_32:
    push edx
    push eax
.wait:
    mov dx, COM1 + 5
    in al, dx
    test al, 0x20
    jz .wait
    pop eax
    mov dx, COM1
    out dx, al
    pop edx
    ret

serial_puts_32:
    lodsb
    test al, al
    jz .done
    call serial_putc_32
    jmp serial_puts_32
.done:
    ret

BOOT_DRIVE: db 0
MSG_BOOT_OK: db 'BOOT_OK', 0x0A, 0
MSG_BOOT_DISK_ERROR: db 'BOOT_DISK_ERROR', 0x0A, 0

gdt_start:
    dq 0x0000000000000000

gdt_code:
    dw 0xFFFF
    dw 0x0000
    db 0x00
    db 10011010b
    db 11001111b
    db 0x00

gdt_data:
    dw 0xFFFF
    dw 0x0000
    db 0x00
    db 10010010b
    db 11001111b
    db 0x00

gdt_end:

gdt_descriptor:
    dw gdt_end - gdt_start - 1
    dd gdt_start

CODE_SEG equ gdt_code - gdt_start
DATA_SEG equ gdt_data - gdt_start

times 510-($-$$) db 0
dw 0xAA55
```

Ghi chú:
- BIOS chỉ đảm bảo boot sector được load tại physical `0x7C00` và drive number nằm trong `DL`. Đừng assume `DS`, `ES`, hoặc `SS` đã đúng; init segment registers trước mọi memory/string/disk access.
- Snippet này minh họa contract. Khi implement thật, verify `boot.bin` đúng 512 bytes. Nếu logic vượt sector 0, chuyển sang 2-stage bootloader thay vì nhét thêm code vào first-stage.
- `KERNEL_SECTORS` không nên hardcode trong source. Generate từ size của `build/kernel.bin` để tránh load thiếu kernel rồi crash.
- Phase-1 loader hiện dùng BIOS CHS read (`INT 13h AH=02h`) theo từng sector và roll qua sector/head/cylinder trên floppy geometry 1.44MB. Contract hiện tại giới hạn `KERNEL_SECTORS <= 120`; nếu kernel lớn hơn, đổi sang LBA extended read (`INT 13h AH=42h`) hoặc 2-stage loader.

### 3. Kernel Entry (`entry.asm`)

```nasm
[bits 32]
section .entry
[extern kernel_main]

global _start
_start:
    mov ebp, 0x90000
    mov esp, ebp

    call kernel_main

    cli
    hlt
```

### 4. Kernel Main (`kernel.c`)

```c
#include "serial.h"
#include "vga.h"

void kernel_main(void) {
    serial_init();
    serial_puts("KERNEL_INIT_OK\n");

    vga_init();
    vga_puts("Kernel initialized successfully!\n");

    while (1) {
        __asm__ volatile("hlt");
    }
}
```

## Marker Rules

- `BOOT_OK` comes from bootloader COM1 output after disk read returned success and protected mode was reached.
- `BOOT_OK` is a loader-progress marker, not proof of kernel identity. If the project adds a kernel header/magic, verify it before emitting a stronger marker.
- `KERNEL_INIT_OK` comes from kernel COM1 output after `serial_init()`.
- `SHELL_READY` is optional until the shell exists; in the current shell phase it is required and must follow `shell_init()`.
- BIOS/VGA text is useful for human debug, but automated tests read `build/serial.log`.

## Common Issues

| Issue | Triệu chứng | Nguyên nhân | Fix |
|---|---|---|---|
| No bootable device | QEMU không boot | `boot.bin` không flat 512 bytes hoặc thiếu `0xAA55` | Build `boot.asm` with `nasm -f bin` |
| Chỉ có `BOOT_OK` | Kernel không chạy tới `kernel_main()` | Load wrong sectors, GDT/stack/jump bug | Check `kernel.bin`, sector count, protected mode |
| Không có `BOOT_OK` nhưng VGA có text | Marker in ra VGA/BIOS thay vì COM1 | Test harness không capture VGA | Emit marker through COM1 |
| Triple fault | QEMU reboot/exit | GDT, segment registers, stack, or bad jump | Run debug mode and inspect transition |
| `BOOT_DISK_ERROR` | Bootloader không đọc được kernel | Disk geometry/sector count/image layout wrong | Check `dd seek=1` and `KERNEL_SECTORS` |
| Random string/disk failures | `DS`/`ES`/`SS` undefined at boot | BIOS does not promise usable data/stack segments | Initialize real-mode segments before memory access |
| Kernel partially loads | Sector count too small | `kernel.bin` grew beyond `KERNEL_SECTORS` | Generate `boot_config.inc` from kernel size |
| Kernel exceeds 120 sectors | Phase-1 CHS profile too small | Loader profile no longer fits kernel size | Implement LBA or 2-stage loader |
